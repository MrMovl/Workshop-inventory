02 — Components

What changed, atom by atom

Same set of primitives the app already uses, restyled. Sizes are calibrated to a 48px primary touch target.

Button — primary
Ink fill on paper. The default action across the app.

Recessed paper.2 well, 1px line border, ink text. Focus ring is ink at 6% over ink border.
bg: paper.2 · border: line · radius: 10 · pad: 12×14 · font: 16

04 — Handoff

Drop-in for Claude Code

One new file (src/theme.ts), one tweak per existing screen. No dependencies, no behavior changes — only StyleSheet values are touched. Paste this whole section into Claude Code with: "Apply the visual direction in this doc. Don't change behavior or props."

Files

+ src/theme.ts (new) App.tsx src/navigation/AppNavigator.tsx src/screens/HomeScreen.tsx src/screens/AddEditBoxScreen.tsx src/screens/AddEditItemScreen.tsx Fonts (one-time)
What this is. A pure restyle. Every JSX tree, prop, hook, and nav route stays identical. Only color/font/spacing values change. Safe to ship behind a single PR.

Fonts — one-time setup Optional

If you don't already load custom fonts, install @expo-google-fonts/inter and @expo-google-fonts/jetbrains-mono and gate render on useFonts(...). If you'd rather skip fonts and stay system, drop the fontFamily values from the theme — every other token still works.

// App.tsx — top of component
import { useFonts, Inter_400Regular, Inter_500Medium, Inter_600SemiBold } from '@expo-google-fonts/inter';
import { JetBrainsMono_400Regular } from '@expo-google-fonts/jetbrains-mono';

const [loaded] = useFonts({
  Inter_400Regular, Inter_500Medium, Inter_600SemiBold,
  JetBrainsMono_400Regular,
});
if (!loaded) return null;
Acceptance check. After applying: tap through Home → Add Item → pick a box → Save. The flow should behave identically. Visually: warm paper background, ink-black primary buttons, cedar-amber Save buttons on edit screens, mono uppercase field labels, dotted chips for categories. No blue should remain anywhere in the app.
